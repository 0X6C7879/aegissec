# 项目需求文档（PRD）
**项目名称**：aegissec  
**产品形态**：个人开源版，本地优先  
**版本目标**：MVP / Beta / v1.0

---

## 1. 背景

现有很多 Agent 项目要么：
- 偏研究原型，能展示但不适合个人长期维护
- 偏企业平台，架构过重
- 偏某个生态，无法复用 Claude Code / OpenCode / 通用 Agent 的 Skill 与 MCP 资产

本项目希望做一个更适合个人维护和开源传播的版本：

- 用 Python 快速构建后端
- 用 Web UI 统一交互和可视化
- 用 Kali Docker 作为受控执行环境
- 兼容主流 Agent 生态中的 Skill 与 MCP 资产
- 保持架构简洁、部署简单、后续可扩展

更具体地说，`aegissec` 的最高任务不是做一个泛化 Agent 壳，而是构建一个**面向授权环境的防御型安全研究工作台**，优先服务以下场景：

- SRC 场景中的自动化众测与主流漏洞发现
- 典型 CVE、云安全与 AI 基础设施问题的可复现验证
- 多层网络与 OA 环境中的多步攻击路径规划、权限维持分析与证据串联
- 基础域渗透实验环境中的企业核心内网推演

因此，产品能力的取舍标准应优先看它是否有助于**验证、建模、留痕与报告**，而不是是否让系统更像一个泛化攻击平台。

---

## 2. 产品目标

### 2.1 核心目标
构建一个单机可运行、面向授权环境的防御型安全研究工作台，使用户可以围绕漏洞验证、攻击路径分析和证据输出完成闭环：

1. 在 Web 中发起会话并与 Agent 对话  
2. 让 Agent 按预定义 Workflow 组织任务  
3. 在受控 Kali 容器中执行动作并采集日志  
4. 识别本地或项目中的通用 Skills  
5. 导入并控制 MCP servers  
6. 实时查看任务图与证据因果图  
7. 生成结构化报告  

### 2.2 最高优先级任务
- 支持 SRC 场景下的自动化众测与主流漏洞发现
- 支持典型 CVE、云安全与 AI 基础设施问题的可复现验证
- 支持多层网络与 OA 环境中的多步攻击路径规划、关键节点验证与权限维持分析
- 支持基础域渗透实验环境中的企业核心内网推演

### 2.3 辅助目标
- 适合个人维护
- 尽量复用第三方库
- 能兼容 Claude Code / OpenCode / 通用 Agent 生态
- 未来可平滑升级到更强架构

## 3. 用户画像

### 3.1 主要用户
- 处理 SRC、众测与日常漏洞验证任务的个人安全研究者
- 需要验证典型 CVE、云安全与 AI 基础设施问题的研究者
- 需要在多层网络、OA 或基础域实验环境中做推演与证据整理的个人维护者
- 需要本地研究工作台的开发者与开源 Agent 作者

### 3.2 使用环境
- macOS / Linux / Windows + WSL
- 本地 Docker Engine
- 单用户为主

## 4. 用户故事

本项目的用户故事必须服务于以下结果：能够把授权环境中的研究任务稳定收敛为“计划—收集—验证—归因—报告”的闭环，而不是停留在零散对话或无约束动作执行。

### 4.1 对话与会话
- 作为用户，我希望在 Web 中创建多个研究会话并切换
- 作为用户，我希望看到会话的任务状态、日志和图谱

### 4.2 Skill 识别
- 作为用户，我希望项目能识别仓库根目录 `skills/` 中的 Skill
- 作为用户，我希望在 UI 中看到 Skill 名称、描述、来源和兼容性
- 作为用户，我希望重新扫描 Skill 目录

### 4.3 MCP 控制
- 作为用户，我希望导入 Claude/OpenCode 的 MCP 配置
- 作为用户，我希望在 Web 中启用或禁用某个 MCP server
- 作为用户，我希望看到该 server 暴露了哪些 tools/resources/prompts

### 4.4 Workflow
- 作为用户，我希望 Agent 按固定流程推进，而不是完全自由发散
- 作为用户，我希望可以看到任务图和当前阶段
- 作为用户，我希望在关键动作前设置审批点

### 4.5 Runtime
- 作为用户，我希望所有执行动作都在 Kali 容器里进行
- 作为用户，我希望能看到 stdout / stderr / exit code
- 作为用户，我希望工件可以被保存与回看

### 4.6 报告
- 作为用户，我希望一键导出 Markdown / HTML 报告
- 作为用户，我希望报告里包含发现、证据、任务历史与结论链

## 5. 功能需求

## 5.1 会话管理
### 必须
- 创建会话
- 删除会话
- 重命名会话
- 查看历史会话
- 恢复会话
- 会话状态显示（idle/running/paused/error/done）

### 可选
- 收藏会话
- 会话标签
- 搜索会话

## 5.2 对话系统
### 必须
- 用户输入
- Agent 回复
- 流式输出
- 事件流（任务启动、阶段切换、执行结果）
- 支持附件元数据引用

### 可选
- Markdown 渲染
- 代码块复制
- 工件直接预览

## 5.3 Workflow 引擎
### 必须
- 定义阶段化 Workflow
- 支持任务节点状态管理
- 支持依赖关系
- 支持暂停 / 继续
- 支持审批节点
- 支持失败回退到上一步

### 可选
- 多模板切换
- 条件分支
- 会话级自定义 Workflow

## 5.4 Skills 兼容
### 必须
- 扫描项目和用户路径
- 解析 `SKILL.md`
- 解析 YAML frontmatter
- 展示名称 / 描述 / 目录 / 来源 / scope / compatibility
- 记录无效 Skill 的错误原因

### 可选
- Skill 内容预览
- 按来源筛选
- 按名称搜索

## 5.5 MCP 兼容
### 必须
- 导入 Claude `.mcp.json`
- 导入 Claude `~/.claude.json`
- 导入 OpenCode `opencode.json`
- 展示 server 列表
- 展示启停状态
- 支持启用 / 禁用
- 发现 tools/resources/prompts
- 显示 server 的 transport / source / scope
- 记录错误信息

### 可选
- 认证状态展示
- 重试连接
- 连通性测试

## 5.6 Runtime
### 必须
- Kali 容器启动与复用
- 工作目录挂载
- 命令执行
- 超时控制
- stdout/stderr 采集
- exit code 记录
- 工件路径登记

### 可选
- 多 Runtime Profile
- 容器快照
- 一键清理工作区

## 5.7 图谱
### 必须
- 任务图
- 因果图
- 节点点击查看详情
- 图谱随任务更新而推送到前端

### 可选
- 证据图
- 图谱筛选
- 回放历史版本

## 5.8 报告
### 必须
- Markdown 导出
- HTML 导出
- 包含会话摘要
- 包含发现和证据引用
- 包含任务图摘要

### 可选
- PDF 导出
- 自定义模板
- 报告比对

---

## 6. 非功能需求

## 6.1 易部署
- 单机部署优先
- 首选 Docker Compose
- 数据库存储为 SQLite

## 6.2 可维护
- 代码结构清晰
- 模块边界明确
- 优先依赖成熟库
- 有基础类型检查和测试

## 6.3 可观测
- 结构化日志
- MCP 错误留痕
- Runtime 执行留痕
- 会话事件可回看

## 6.4 性能
- 单用户场景下，普通会话 UI 操作响应 < 500ms
- Skill 重扫在中小型仓库内保持可接受
- MCP 能力发现结果可缓存

## 6.5 安全
- Runtime 默认受限
- 审批门禁
- 敏感配置不直接暴露到 UI
- 本地配置文件权限提示

## 7. 兼容需求

## 7.1 Skill 目录兼容
### Skills
- 识别仓库根目录 `skills/**/SKILL.md`
- 兼容基础 frontmatter 字段
- 允许保留未知字段
- 展示统一来源 `local` 与项目级 scope

### MCP
- 识别 `.mcp.json`
- 识别 `~/.claude.json`
- 识别本地 stdio 与远程 HTTP 配置

## 7.2 OpenCode 兼容
### Skills
- 支持将 OpenCode Skill 复制到仓库根目录 `skills/` 后进行识别

### MCP
- 识别 `opencode.json` 中的 `mcp`
- 兼容 `enabled`
- 兼容 local / remote 类型

## 7.3 通用 Agent 兼容
- 为未知 frontmatter 做透传保留
- 使用统一元数据模型进行展示

## 9. 成功标准

### MVP 成功标准
- 新用户可在 30 分钟内本地跑起项目
- 能导入至少一种 Claude MCP 配置和一种 OpenCode MCP 配置
- 能识别至少三种 Skill 目录来源
- 能看到可用 Skills 与 MCP servers
- 能跑完一条完整 Workflow 并生成报告
- 能围绕至少一个代表性验证/推演场景沉淀出可复现的证据与结构化输出
- 能确保系统整体仍以授权、受控、可留痕为边界，而不暗示无约束攻击自动化

### Beta 成功标准
- MCP 连接更稳定
- 文档完备
- 有示例配置
- 有基础自动化测试

## 10. 验收标准

### 10.1 Skill 兼容验收
- 给定 `skills/demo/SKILL.md`，系统能识别并显示
- 当项目中同时存在 `.claude/.opencode/.agents` 兼容目录时，系统仍只读取 `skills/`
- 非法 frontmatter 可显示错误原因

### 10.2 MCP 兼容验收
- 导入 `.mcp.json` 后 UI 中能展示 server
- 导入 `opencode.json` 后 UI 中能展示 server
- 能启用 / 禁用 server
- 能看到 tools/resources/prompts 数量

### 10.3 Workflow 验收
- 新建任务后，能进入“范围确认 → 环境启动 → Skill/MCP 同步 → 收集 → 验证 → 报告”流程
- 前端图谱与任务状态同步更新

### 10.4 Runtime 验收
- 能成功启动 Kali 容器
- 能在容器中执行命令
- 能获取 stdout/stderr/exit_code
- 能记录工件路径

## 11. 风险与约束

### 风险
- Claude / OpenCode 配置格式未来变化
- MCP server 各家实现质量不一致
- Skill frontmatter 差异导致兼容成本上升
- 个人维护者时间有限

### 约束
- 所有开发任务都应优先贴合四类核心场景，而不是脱离场景扩张功能面
- 个人版不做复杂分布式架构
- 不重新设计 Skill / MCP 标准
- 必须控制依赖和模块复杂度
- 必须保持授权、受控、可审计的产品边界

---

## 12. 结论

本项目的 PRD 关键点可以概括为一句话：

**aegissec 是一个面向授权环境的本地优先防御型安全研究工作台，围绕漏洞验证、攻击路径分析与证据驱动报告，逐步支持 SRC、典型 CVE / 云安全 / AI 基础设施、多层网络 / OA 与基础域渗透模拟等核心场景。**
