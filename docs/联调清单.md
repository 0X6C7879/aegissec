# 联调清单

本文档用于模块 E 的浏览器验收与接口联调，默认假设本地服务已启动在：

- Web：`http://127.0.0.1:5173`
- API：`http://127.0.0.1:8000`

## 1. 启动前检查

- [ ] `cp .env.example .env`
- [ ] `python scripts/dev.py` 或 `docker compose up --build`
- [ ] `python scripts/seed_demo.py`
- [ ] `http://127.0.0.1:8000/api/health` 返回 200
- [ ] `http://127.0.0.1:8000/docs` 可访问

## 2. Workspace 主路径

- [ ] 打开 `/sessions`
- [ ] 点击“新建对话”
- [ ] 发送一条消息
- [ ] 页面出现用户消息与助手消息
- [ ] 左侧会话列表可见新增会话
- [ ] 若启动 workflow，图谱面板可切换任务图 / 证据图 / 日志

## 3. Graph / History

- [ ] 进入 `/sessions/{sessionId}/graph`
- [ ] 任务图可加载
- [ ] 证据图或因果图可切换
- [ ] 打开 `/history`
- [ ] 回放与导出入口可见

## 4. Projects

- [ ] 打开 `/projects`
- [ ] 创建一个项目
- [ ] 项目详情可看到关联 sessions

## 5. Runtime

- [ ] 打开 `/runtime`
- [ ] 健康状态可加载
- [ ] 最近 runs / artifacts 列表可加载
- [ ] 至少验证一次 start / stop 或 execute 主流程

## 6. Skills / MCP

- [ ] 打开 `/skills`
- [ ] 扫描结果可见
- [ ] 单个 skill 详情可打开
- [ ] 打开 `/mcp`
- [ ] 导入或刷新后可看到 server 列表
- [ ] server 详情、能力或健康状态可查看

## 7. Settings

- [ ] 打开 `/settings`
- [ ] 主题、UI 密度、模型/API 设置可加载
- [ ] 修改后刷新页面仍能保持本地状态

## 8. 接口抽查

- [ ] `GET /api/sessions`
- [ ] `GET /api/projects`
- [ ] `GET /api/runtime/status`
- [ ] `GET /api/skills`
- [ ] `GET /api/mcp/servers`
- [ ] `GET /api/workflows/templates`

## 9. 通过标准

- [ ] 浏览器主路径无阻断错误
- [ ] 控制台无新的关键错误
- [ ] 后端健康检查通过
- [ ] 文档中的命令、端口、路径与实际行为一致
