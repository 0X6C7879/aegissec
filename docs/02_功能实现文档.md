# 功能实现文档
**目标**：把架构设计落到可以直接开工的工程实现层面  
**重点**：模块划分、数据结构、接口、关键流程、依赖选型

---

## 1. 实现总览

V1 采用：

- 后端：FastAPI 单体
- 前端：React SPA
- 存储：SQLite
- 编排：LangGraph
- 运行时：Docker SDK + Kali 容器
- Skill 兼容：目录扫描 + YAML 解析
- MCP 兼容：配置导入 + official Python SDK 客户端
- 图谱：普通表 + JSON + Web 前端渲染

## 5. Skill 兼容实现

## 5.1 扫描器
`compat/skills/scanner.py`

职责：
- 固定使用仓库根目录 `skills/` 作为扫描路径
- 递归查找 `*/SKILL.md`
- 标记来源为项目本地 `local`
- 计算文件哈希
- 输出原始候选列表

### 伪代码
```python
def discover_skill_files(paths: list[Path]) -> list[SkillFileCandidate]:
    results = []
    for root in paths:
        for file in root.rglob("SKILL.md"):
            results.append(build_candidate(file))
    return results
```

## 5.2 解析器
`compat/skills/parser.py`

职责：
- 解析 frontmatter
- 校验名称与目录一致性
- 提取描述、compatibility、metadata
- 保留 `raw_frontmatter`

### 推荐库
- `python-frontmatter` 或 `ruamel.yaml`

## 5.3 Skill Service
`compat/skills/service.py`

职责：
- rescan
- diff（新增/变更/失效）
- 数据库存储
- 向 UI 返回统一模型

### 对外接口
- `rescan_skills()`
- `list_skills()`
- `get_skill(skill_id)`
- `read_skill_content(skill_id)`

## 6. MCP 兼容实现

## 6.1 导入器
`compat/mcp/importer.py`

职责：
- 识别输入来源
- 解析 Claude `.mcp.json`
- 解析 Claude `~/.claude.json`
- 解析 OpenCode `opencode.json`
- 转成统一 `MCPServerConfig`

### 统一模型
```python
class MCPServerConfig(BaseModel):
    name: str
    source: Literal["claude", "opencode", "custom"]
    scope: Literal["project", "user", "system"]
    transport: Literal["stdio", "http", "sse"]
    enabled: bool = True
    command: list[str] | None = None
    args: list[str] | None = None
    env: dict[str, str] = {}
    url: str | None = None
    headers: dict[str, str] = {}
    timeout_ms: int = 5000
```

## 6.2 Client Manager
`compat/mcp/client_manager.py`

职责：
- 建立 server 连接
- 保存 active sessions
- 管理启停
- 触发能力发现
- 包装 tool 调用

### 推荐方式
基于 official `mcp` Python SDK 的 client/session group 封装：
- 对 stdio server 用受控子进程
- 对 remote server 建立 HTTP 会话

## 6.3 Discovery
`compat/mcp/discovery.py`

职责：
- 拉取 `tools`
- 拉取 `resources`
- 拉取 `prompts`
- 写入数据库
- 生成 UI 展示摘要

## 6.4 MCP Service
`compat/mcp/service.py`

对外接口：
- `import_from_file(path)`
- `list_servers()`
- `toggle_server(server_id, enabled)`
- `refresh_capabilities(server_id)`
- `call_tool(server_id, tool_name, arguments)`

## 7. Kali Runtime 实现

## 7.1 Docker Manager
`runtime/docker_manager.py`

职责：
- 检查镜像
- 创建容器
- 启动 / 停止 / 删除
- 绑定工作目录
- 限制 CPU / 内存 / 网络策略（按 profile）

### 推荐库
- `docker`（Docker SDK for Python）

## 7.2 Executor
`runtime/executor.py`

职责：
- 接收命令
- 在容器内执行
- 返回 stdout/stderr/exit_code
- 支持 timeout
- 记录执行历史

### 返回结构
```python
class ExecutionResult(BaseModel):
    run_id: str
    status: Literal["success", "failed", "timeout"]
    exit_code: int | None
    stdout: str
    stderr: str
    started_at: datetime
    ended_at: datetime
    artifacts: list[str] = []
```

## 7.3 Artifact Service
职责：
- 收集执行产生的文件
- 写入 `artifact` 表
- 支持下载或预览

## 8. Workflow 引擎实现

## 8.1 模板
工作流模板放在 `config/workflows/*.yaml`。

示例：
```yaml
name: authorized-assessment
stages:
  - key: scope_guard
    title: 范围确认
  - key: runtime_boot
    title: 环境启动
  - key: skill_mcp_sync
    title: Skill/MCP 同步
  - key: context_collect
    title: 上下文收集
  - key: hypothesis_build
    title: 假设生成
  - key: safe_validation
    title: 低风险验证
  - key: findings_merge
    title: 发现归并
  - key: causal_graph_update
    title: 因果图更新
  - key: report_export
    title: 报告导出
```

## 8.2 状态模型
推荐使用 LangGraph state：

```python
class WorkflowState(TypedDict):
    session_id: str
    current_stage: str
    messages: list[dict]
    skill_snapshot: list[dict]
    mcp_snapshot: list[dict]
    findings: list[dict]
    graph_updates: list[dict]
```

## 8.3 节点实现
每个阶段一个 node function：
- `scope_guard_node`
- `runtime_boot_node`
- `skill_mcp_sync_node`
- `context_collect_node`
- `hypothesis_build_node`
- `safe_validation_node`
- `findings_merge_node`
- `causal_graph_update_node`
- `report_export_node`

## 8.4 事件推送
每个阶段结束后：
- 写库
- 发送 WebSocket 事件
- 更新图谱

## 9. 图谱实现

## 9.1 为什么不单独上图数据库
V1 图规模不大，且单用户场景下：
- SQLite + JSON 足够
- 开发更快
- 部署更简单

## 9.2 图模型
统一节点结构：
```python
class GraphNodeDTO(BaseModel):
    id: str
    graph_type: Literal["task", "evidence", "causal"]
    node_type: str
    label: str
    data: dict = {}
```

统一边结构：
```python
class GraphEdgeDTO(BaseModel):
    id: str
    graph_type: Literal["task", "evidence", "causal"]
    source: str
    target: str
    relation: str
    data: dict = {}
```

## 9.3 任务图
使用 React Flow 渲染：
- status 决定节点颜色
- current_stage 增加高亮
- 支持点击查看任务详情

## 9.4 因果图
使用 Cytoscape.js 渲染：
- supports / contradicts / validates 不同边样式
- 节点侧边栏展示证据详情

## 17. 结论

V1 的功能实现重点不是“把所有能力做全”，而是把下面这个闭环做通：

**会话创建 → Skill/MCP 识别 → Workflow 运行 → Kali 执行 → 图谱更新 → 报告导出**
