# Demo 视频 / 浏览器演示脚本

## 1. 演示目标

在 5~8 分钟内展示 AegisSec 的最小可用闭环：

1. 创建 / 打开会话
2. 发送消息
3. 查看图谱与历史
4. 查看 Runtime / Skills / MCP / Settings 页面
5. 证明文档与项目结构自洽

## 2. 演示前准备

```bash
cp .env.example .env
python scripts/seed_demo.py
python scripts/dev.py
```

浏览器打开：`http://127.0.0.1:5173`

## 3. 分镜脚本

### 镜头 1：主工作台

- 打开 `/sessions`
- 说明左侧是会话和任务树，中间是对话，右侧是图谱与日志
- 点击“新建对话”并发送一条测试消息

讲解词：

> 这里是统一的 Workspace，用户不需要在多个低频后台页面之间来回切换。

### 镜头 2：历史与图谱

- 进入当前 session 的 graph 页面
- 切换任务图 / 证据图 / 日志
- 打开 `/history`，说明支持回放与导出

讲解词：

> 会话执行过程不是一次性输出，而是可追踪、可回放、可导出的。

### 镜头 3：Runtime

- 打开 `/runtime`
- 展示运行时健康状态与最近执行记录

讲解词：

> Runtime 页面用于验证执行环境、工件和命令记录，便于审计和复盘。

### 镜头 4：Skills / MCP

- 打开 `/skills` 展示技能扫描结果
- 打开 `/mcp` 展示 MCP server 列表和能力信息

讲解词：

> 平台可以兼容 Skills 和 MCP，两者都通过统一页面进行可见化管理。

### 镜头 5：Settings 与文档

- 打开 `/settings`
- 展示主题/密度等本地设置
- 回到代码仓库说明 `README.md`、`docs/API约定.md`、`docs/联调清单.md`

讲解词：

> 模块 E 的目标不是只把功能跑起来，而是把工程规范、文档、测试和验收链路一起补齐。

## 4. 演示完成标准

- 浏览器无关键报错
- 主路径页面都能打开
- 文档中命令、端口和页面路径与实际一致
